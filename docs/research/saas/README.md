# SaaS Strategy Research

Decision-grade research on productizing the MetaHarness/Darwin stack as a commercial service.

## Documents

- [**fugu-clone-orchestrator.md**](./fugu-clone-orchestrator.md) — Strategy & feasibility for a transparent, self-optimizing orchestrator on GCP (a governable "Fugu-clone"). Synthesizes 7 deep-research streams: competitive analysis of Sakana Fugu, GCP architecture, Darwin-as-conductor, cost-economics, vertical-vs-general positioning, the governance moat, and go-to-market packaging. Includes a concrete thin-slice MVP build plan.

## TL;DR

Build a **vertical, audit-trail-native, cost-transparent** orchestrator — same cheap-base + frontier-escalation architecture this repo already validated (LEARNINGS §30), but exposed as an inspectable, lineage-audited pipeline rather than Fugu's opaque black box. Compete on the customer's own tasks (CI/CD, FinOps), lead with cost-Pareto + EU AI Act governance, and never claim a leaderboard number we cannot reproduce. The hard-tail ceiling (~47–62% clean) is a shared model-reasoning limit orchestration cannot route past — state it plainly.
